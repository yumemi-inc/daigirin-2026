# 中古パソコン漁りから始めるモバイルアプリ開発

モバイルアプリを開発するために Mac を買う、この構図は長らく変わっていません。この機材購入は、開発を始める準備の第一歩であり、もっとも難しいことの１つです。

毎年 MacBook Air を買い替えれば安く最新マシンを運用できるというのは、とうの昔の話。現在の Apple 製品は、PC 市場の中でも高価格帯のカテゴリに属しています。為替や世界情勢といった外的要因により、開発用途として快適なスペックを求めると、価格が上がりやすいです。加えて、オンボードのメモリやストレージにより、あとからユーザーが増設できないため、数年後の開発状況を見据えて、購入時にスペックを盛る場面もあります。

また 2025 年末から PC 市場全体は、メモリを始めとしたパーツ価格が極端に上昇しています。現時点では Apple 製品への影響は現れていませんが、高価になるリスクは十分に存在します。

開発における「最初に Mac を購入する」という準備が、年々難しくなっています。社会人は会社から貸与されるのでマシとして、もっとも影響があるのは、これから開発を始める学生さん達です。開発そのものが、大きな負担になるでしょう。

## iOS アプリの開発フローの整理と依存関係

Mac が必須となる開発として、iOS アプリの開発を対象とします。iOS アプリ開発は Mac が不可欠です。しかし、この状況も、一部が置き換えもできるようになりました。

|フェーズ|依存先|非Mac代替の可否|
|:--|:--|:--|
|開発|エディタ選択の自由|可能|
|ビルド|Apple ツールチェイン|一部可能|
|署名|Apple Developer Program|可能（CLIツールあり）|
|配布|Apple公式のアップロード手段|CIで代替可能|

特にビルド工程は、Apple 純正ツール（Xcode の内部 API）への依存度が高いため、完全な代替は難しい領域です。しかし近年登場したサービスにより、非 Mac 環境からビルドを委任できる形が整いつつあります。Mac は「絶対必須」ではなく、「最終段階で必要になる」の存在になったともいえるでしょう。

### なぜ非 Mac 開発環境を考えるべきなのか

Mac だけが選択肢である状況は、開発者の裾野を狭めます。特に、学生や初学者で iOS 開発を始めたい人にとっては、低コストで始められることは継続性に直結します。ツールの進化により、「Mac がないと何もできない」ということは過去になりつつあります。

なお、私は iOS アプリ開発において、Mac 環境下で開発するのが一番よいと考えます。その開発体験を犠牲にして、他プラットフォームで開発する意味があるのかと考えるでしょう。開発体験をよりも、そもそも「自身がアプリを作った」という成功体験を浴びてほしいです。Mac をお勧めしますが、まずは自身の手を動かして、何かを作るということを経験して欲しいです。

### 本書の目的と読者が得られること

本書では、実際に Mac を使わずに iOS アプリを開発する方法を紹介します。

- 非 Mac 環境での開発環境の構築方法
- 中古 PC を活用した低コスト開発環境の構築

本書のゴールは Mac がないからと、開発を諦めてた意識を変えることです。

## 非 Mac 環境でアプリ開発

非 Mac ユーザーが iOS アプリ開発をする場合、現在もっとも現実的な選択は React Native や　Flutter といったクロスプラットフォームの採用です。ただし、iOS アプリをビルドする場合は、通常は Mac が必要になります。React Native は Expo を利用して、Expo 上でビルドしたら、Mac は不要です。また、GitHub Actions や Bitrise など　CI サービスを利用して、CI 上の Mac でビルドする方法もあります。

クロスプラットフォームは便利ですが、「Swift を学びたい」と考える人にとっては、不向きです。本節では、Windows や Ubuntu といった Mac 以外の OS を利用して、ネイティブの iOS アプリを作るための、現代的なアプローチを整理します。

### XTool を利用した Ubuntu 環境での iOS アプリ開発

「Mac はないけれど、純粋な Swift と iOS SDK を使って開発したい」、そんな硬派なニーズに応えるのが、XTool [^xtool] です。XTool は、Windows や Ubuntu 上で、iOS　向けのネイティブアプリをビルドし、署名まで行えるオープンソースのツールチェインです。

[^xtool]: xtool, https://github.com/xtool-org/xtool

<!-- 実績報告例: https://qiita.com/softbase/items/7bdfa75aa88a649f57fb -->

本来、Swift 言語自体はオープンソースであり Linux などでも動作します。しかし、Linux 上の Swift で iOS 用のアプリを作るには、Apple が提供するフレームワーク（UIKit など）などの利用が必要です。XTool は、この環境構築を非 Mac 環境上で実現します。

クロスプラットフォーム開発が「別の言語で書いて変換する」アプローチだとすれば、XTool は「純正の Swift コードを、Linux 上で iOS 向けに翻訳する」アプローチです。なお、Xcode 自体にある開発機能は利用できません。その便利な機能とトレードオフで、開発できます。

## XTool

XTool は、2025 年に登場した「Mac なしで iOS アプリを開発・ビルド・実機インストールできる」オープンソースのツールチェインです。 従来の「iOS アプリをビルドして iPhone で動かすには Mac が必須」という壁を破壊した画期的なツールです。

### XTool とは？（簡単な説明）

XTool は、Xcode の機能をコマンドライン（CLI）で再現したクロスプラットフォームツールです。 XTool は Apple から提供される Xcode.xip（インストール用ファイル）から必要な SDK だけを抽出し、独自のビルドシステムを使ってアプリを生成します。これにより、Ubuntu や Windows（WSL）上で、SwiftPM（Swift Package Manager）ベースの iOS アプリを作成します。また、USB 接続した iPhone にアプリをインストールできます。

### 導入方法（Ubuntu環境）

セットアップ方法は公式サイト [^xtool-install] を参照してもらいたいですが、簡単に書くと次のとおりです。

[^xtool-install]: https://xtool.sh/documentation/xtool/installation-linux/

1. Swift をダウンロードする
1. Xcode.xip をダウンロードする
1. App Store Connect API で App Manager 権限の Team Key の API Key を発行する
1. XTool をインストールをする
1. ダウンロードした　Xcode.xip と API Key を使って、XTool をセットアップする

Ubuntu に iPhone を有線接続して、次のコマンドを打つと、iOS アプリがビルドされて、iPhone にインストールされます [^xtool-first-app]。

[^xtool-first-app]: https://xtool.sh/tutorials/xtool/first-app

```shell
xtool new Hello // プロジェクトの作成
cd Hello        // プロジェクトディレクトリへ移動
xtool dev       // プロジェクトのビルドとインストール
```

実際に業務に使えるかは別として、Ubuntu で iOS アプリが作成できるのは驚きました。XTool は Linux 環境で Swift を学びたい人にとって、最適な選択肢です。これまで不可能だった「Linux での iOS 実機開発」を現実のものにしました。

## 開発できる安い PC を探して

非 Mac 環境で iOS アプリ開発を始めるにあたり、最初に直面する現実的な壁は PC 本体の値段です。開発に耐えうるスペックといえば、メモリ 16GB 以上、SSD 250GB 以上ぐらいでしょうか。このスペックの新品の PC を購入しようとすれば、Mac ほどではないにせよ、Windows でも 10 〜 15 万円ほどの出費は覚悟しなければなりません。国内メーカーなら、もっと高くなるでしょう。「コストを抑えるために Mac を使わない」という選択をしたのに、代わりの PC が高価であっては本末転倒です。

OS 代分を安くするため、そもそも開発機に最適な Ubuntu などを選択するもあります。しかしながら、国内で販売されている PC で Ubuntu が最初からインストールされて販売されている PC はほとんどありません。一時期 Dell が個人向けで販売していましたが、いまはビジネスしかもプロ向けに Ubuntu モデルがあるだけです。また、BTO メーカーの OS なしモデルも同様に見当たらないです。

そこで、私が勧めるのは「中古パソコン」です。ここから本編です。

日本の PC 市場には、企業が 3 〜 4 年のリース契約を終えた高品質なビジネスノート PC（ThinkPad、Latitude、VersaPro など）が大量に還流してきます。これらは元々の定価が 20 万円を超えるような堅牢なモデルであり、それが数万円台という破格の値段で入手可能です。

最近は、M1 Pro を搭載した MacBook Pro の中古が市場に多く現れています。リリースから 4 年以上経過したので、市場に流れ始めたのでしょう。ただし、Mac は中古でも高いので、今回は取り上げません。もし手軽なら価格で入手できたら、それで開発をしてください。

また、近年のハードウェアは標準化がすすみ、Ubuntu などの Linux OS 側をインストールしても問題なく動作する機種が多いです。発売から数年経過した中古 PC は、世界中の開発者によって動作検証されており、トラブルシューティング情報も豊富です。「枯れた技術（安定した技術）」を安く手に入れることができます。

## 私的に勧めるスペック

中古なら何でもいいわけではありません。快適に開発作業を行うためには、絶対に譲れない私的ラインが存在します。結論からいえば、狙うべきスペックは「Intel 第８世代 Core i5 以上 / メモリ 16GB 以上 / 解像度 FHD」です。

### Intel 第８世代 Core i5 以上

Intel 第８世代以降、i5 でも４コアになりました。コア数は正義です。また、第８世代は Windows 11 の対応ラインであるため、需要があります。その分、中古価格にも影響しますが、市場が活発なので、状態のよい機体が市場に溢れているでしょう。

安さを狙うなら、第６~７世代もありですが、個人的には第８世代以降がお勧めです。Windows 11 がインストールできるのもおおきいですし、この世代の機種から USB PD 充電対応が多いです。USB Type-C で充電できるので、AC 電源を心配する必要もありません。

### メモリ　16GB 以上

VS Code を開き、ブラウザで技術書を読み、バックグラウンドでビルドを回す。これらを同時に行うと 8GB では足りないです。最初から 16GB を搭載したモデルを探すか、購入後に自分で増設できるモデルを選びましょう。

メモリが少ない場合は、Lubuntu など軽量なディストリビューションを選択するのもよいです。

### 悩んだら Lenovo ThinkPad

悩んだら、ThinkPad の中から探すのがよいです。もともと人気な機種なので、多くの中古が出回っています。また、保守パーツが、純正や互換品で売っており、自分で修理や改修もできます。

## Linux / Ubuntu のインストール

中古 PC を手に入れたら、開発向けに Ubuntu をインストールします。「OS のインストール」と聞くとハードルが高く感じるでしょう。しかし、現代の Linux はインストーラーが非常に親切になっており、画面の指示にしたがってクリックするだけです。

ただし、インストールする前に１つ、重要な選択があります。それは、どの Linux もしくは Ubuntu を入れるかという問題です。

### スペックに合わせてディストリビューションやフレーバーを選ぶ

まず、どの Linux ディストリビューションを選ぶかですが、特にこだわりがなければ、Ubuntu でよいです。さらに Ubuntu には、基本となる「Ubuntu（無印）」以外に、フレーバーと呼ばれる派生版が存在します。これらは、中身は同じですが、デスクトップ画面の見た目や操作性が異なります。中古 PC を活用する場合、このフレーバー選びが快適さを左右します。

私は、通常の Ubuntu を利用していますが、低スペックのマシン（低電力コア＆メモリ 8GB）には Lubuntu を利用しています。Ubuntu は、インストール用の USB メモリを用意します。インストールする前にお試しで利用できるので、確認してください。

## Chromebook　は可能性の獣


「Chromebookでアプリ開発？ 無理でしょ？」そう思われるのも無理はありません。Chromebookは長らく、教育現場や家庭での「Webブラウジング専用機」として認知されてきました。

しかし、その常識は過去のものです。現在のChromebookは、**「手軽にLinuxが使える、コスパ最強の開発マシン」**へと進化しています。特に予算が限られる学生や、サブ機で開発したい人にとって、ChromebookはMacやWindows中古機と並ぶ、あるいはそれ以上に魅力的な選択肢になり得ます。

本章では、ChromebookをiOSアプリ開発の「母艦」に変える方法と、その意外な実力について解説します。

### なぜ Chromebook なのか

Chromebookを選択肢に入れる最大の理由は、**「最もリスクが低く、手軽にLinux開発環境が手に入るから」**です。

通常、Windows PCにLinux環境（WSLやデュアルブート）を構築しようとすると、BIOS設定やドライバの相性など、初心者にはハードルの高い作業が伴います。しかし、Chromebookは最初からGoogleによって管理された安全なLinux環境（Crostini）が内包されており、スイッチひとつで呼び出すことができます。

さらに、以下のメリットが開発学習を強力に後押しします。

1. 「壊しても大丈夫」な安心感開発の勉強中、コマンド操作を誤ってシステムをおかしくしてしまうことはよくあります。Chromebookなら、Linux部分だけを削除して再作成したり、本体を「Powerwash（初期化）」したりすることで、数分で新品の状態に戻せます。この「失敗できる環境」こそが、初学者にとって最大の武器になります。
2. 圧倒的なコストパフォーマンス新品でも3〜5万円台から購入可能で、バッテリー持ちも優秀です。高価なMacやWindows機を買う予算がない場合でも、中古市場に頼らず、保証のある新品でスタートラインに立てます。
3. 環境の再現性が高いGoogleアカウントに設定が紐付くため、端末を買い替えてもすぐに元の環境に戻れます。「まずは小さく始めて、失敗しながら学びたい」。そんなアジャイルな学習スタイルに最もフィットするのが、Chromebookなのです。

### ChromebookのLinux環境（Crostini）

Chromebookで開発を行うには、**「Linux 開発環境（Crostini）」**を有効にする必要があります。これはChromeOSの中で動く「仮想マシンのようなもの」ですが、OSとシームレスに統合されており、面倒な設定は不要です。

#### Linuxをオンにする手順

1. Chromebookの右下の時計をクリックし、歯車アイコン（設定）を開きます。
1. 左メニューの「詳細設定」＞「デベロッパー」をクリックします。
1. 「Linux 開発環境」の横にある「オンにする」ボタンをクリックします。
1. セットアップウィザードに従い、ユーザー名とディスクサイズを決めます。ディスクサイズは後から変更できますが、開発用なら**最低10GB以上（推奨20GB〜）**を割り当てておきましょう。

インストールが完了すると、ランチャーに「ターミナル」アプリが追加されます。これを開き、黒い画面に文字が打てるようになれば準備完了です。中身は世界で最も普及している「Debian」というLinuxディストリビューションです。

### 開発ツールを揃える

通常のUbuntu と同様に debファイルをダウンロードしてインストールできます。また、Chromebook でも Ubuntu で紹介した Flatpak が利用できます。

### 5.4 Chromebookでの開発体験

前述のUbuntuと同じ開発環境を再現できます。Chromebook が登場したことは、Google が Android Studio が動作して、Androidアプリ開発ができるとアピールしてましたね。

### 5.5 失敗しないChromebookのスペック選び

「安いから何でもいい」わけではありません。開発用途で買うなら、以下のスペック基準を必ず守ってください。

|項目|必須ライン|推奨ライン|解説|
| :-- | :-- | :-- | :-- |
|CPU|Intel N100 | Core i3 (第10世代以降) | 昔のCeleron (N4020など) は遅すぎて開発になりません。また、ARM系CPU（MediaTekなど）は開発ツールの互換性で苦労することがあるため、初心者はIntel製が無難です。|
|メモリ|8GB|16GB|4GBは絶対に避けてください。 LinuxとChromeブラウザを同時に開くとメモリ不足で落ちます。|
|ストレージ|64GB|128GB以上|システムとLinux領域で容量を食います。32GBモデルはOSだけで埋まるため不可です。|

特に重要なのは 「CPUはIntel系」「メモリは8GB以上」 の2点です。ここさえクリアしていれば、3〜4万円台のモデルでも十分に「戦える」マシンになります。

確実なスペックは、Chromebook の中でも Chromebook plus というブランド名が付けられた機種が確実です。ただ、この機種は６ ~ 10万円以上と少し根が張ります。

### 5.6 導入コストと市場価格

Chromebookは「セールの時」が買い時です。

- 3万円〜5万円: Intel N100 / メモリ8GB のモデルが新品で狙えます。Amazonのセールや、メーカー直販サイトのアウトレットをチェックしましょう。
- 2万円台: 中古市場なら、第8世代Core i5を搭載した法人向けモデル（ThinkPad C13 Yogaなど）が見つかることもあります。

MacBookの1/5以下の価格で、新品かつ保証付きの開発機が手に入るのは、Chromebookならではの特権です。

### 5.7 運用上の制約と限界

万能に見えるChromebookですが、制約もあります。

1. Xcodeは動かない: 本書で繰り返し述べている通り、純正のXcodeは使えません。最終的なストアへのアップロードにはCI/CDサービスなどが必須になります。
2. ストレージ管理: Linux環境は「一つの大きなファイル」として扱われるため、空き容量の管理に少し気を使います。動画などの大きなファイルはGoogleドライブや外部SDカードに逃がしましょう。
3. 周辺機器の相性: 特殊なドライバが必要なUSB機器などは、Linux環境側でうまく認識しないことがあります。

### 5.8 Chromebookの活かしどころ

Chromebookは、「メインの最強マシン」にはなれないかもしれません。しかし、「最高のサブ機」、あるいは**「最短で走り出すためのスタート地点」**としては最強です。

- 学生・初学者: 高価なMacを買う前に、「プログラミングが自分に向いているか」を低コストで確かめられます。
- カフェでのコーディング: バッテリー持ちが良く軽量なため、重いMacBook Proを持ち歩かずに、身軽にコードを書きに行けます。
- プロトタイピング: アイデアを思いついたらすぐに開き、Expoでサッと形にしてiPhoneで動かす。その機動力はプロのエンジニアにとっても魅力的です。

### 5.9 本章のまとめ

- Chromebookは、標準機能で安全なLinux環境（Crostini）が使える優秀な開発機である。
- 開発ツールは「Flatpak」を使ってVS Codeなどを導入するのがモダンな手法。
- 開発スタイルは「Expo」が最も快適。スペックが許せば「XTool」も動く。
- 購入時は「Intel CPU (N100以上)」と「メモリ8GB以上」を死守すること。

「Chromebookだからできない」と諦める必要はありません。むしろ、制約があるからこそ工夫が生まれ、Linuxの知識も身につきます。その経験は、将来どんなエンジニアになっても必ず役立つ武器になるはずです。

## おわりに

本書では、これまで「iOS開発＝Mac必須」という強固な固定観念が、技術の進化によって崩れつつある現状を紹介してきました。

WindowsやUbuntu、そしてChromebookであっても、開発は可能です。 クロスプラットフォーム技術（Expo）を使えばOSの壁を越えてコードを書くことができ、XToolのようなツールチェーンを使えばLinux環境からネイティブなビルドを行い、手元のiPhone実機で動かすことさえできます。さらに、CI/CD（継続的インテグレーション）サービスを活用すれば、App Storeへの配信という最後の関門さえも、Macを持たずに突破するルートが開かれています。

「Macを持っていないから、iOSアプリは作れない」。 もしあなたがそう思っていたなら、それはもう過去の常識です。今、あなたの目の前には、Macがなくても走り出せる選択肢が十分に用意されています。

### 8.2 初期投資を抑えることが、挑戦の数を増やす

開発を志す人、特に学生や個人開発者にとって、最も高く立ちはだかる壁は常に「コスト」でした。 「アプリを作ってみたい」という純粋な衝動が生まれたその瞬間に、「まずは20万円のMacを買ってください」と突きつけられる現実は、あまりにも残酷なフィルタリングでした。

本書が提案した「中古PCやChromebookを活用する」というアプローチは、単なる節約術ではありません。 初期投資を数万円レベルまで下げることは、「失敗する権利」を手に入れることと同義です。

もし途中で挫折しても、金銭的なダメージは最小限で済みます。リスクが低ければ低いほど、人は気軽に挑戦できます。開発環境のコストを理由に、未来のクリエイターがその芽を摘まれるようなことはあってはならない。本書の根底には、そのような願いがあります。

### 8.3 「必要になってから備える」というエンジニアリング思考

本書は決して「Mac不要論」を唱えるものではありません。むしろ、プロフェッショナルとして生きていく上で、Macはいつか必要になる素晴らしいツールです。

重要なのは**「いつ投資するか」**というタイミングです。

まずは手元にあるWindows機や、安価な中古PCで開発を始めてみる。 実際にアプリを作り、リリースし、ユーザーからのフィードバックを得る。 そうして「もっと高度なネイティブ機能を使いたい」「ビルド時間を短縮したい」「収益が出てきた」という明確なニーズが生まれた時こそが、Macを導入する最適なタイミングです。

ソフトウェア開発において、不確実な未来のために過剰なリソースを割くことは推奨されません（YAGNI原則）。ハードウェアへの投資も同じです。「必要になった時に、必要な分だけ投資する」。それは極めて合理的で、正しいエンジニアリング思考です。

### 8.4 これからのiOS開発はもっと自由になる

VisionOSの登場、WebGPUの発展、開発ツールへのAI統合など、Appleのエコシステムは広がり続け、開発手法も多様化しています。 特定のハードウェアを持っていなければ参加すらできないという時代は終わり、アイデアと情熱さえあれば、誰でもそのフィールドに立てる土台が整いつつあります。

この変化の時代において、「環境がないから」と立ち止まるのはあまりにも勿体無いことです。 PCが手元にあり、iPhoneがあり、そして「作りたい」という気持ちがある。それだけで、条件はすべて揃っています。

### 8.5 最後に

あなたがこの本を手に取った理由が、「自分の手でアプリを作ってみたい」。 たったそれだけの、しかし熱い思いからだったとしたら、あなたはすでにスタートラインを越えています。

Macがなくても、始めることはできます。 始めさえすれば、必ず次の景色が見えてきます。 その景色を、ぜひあなた自身の手で切り拓いてください。

本書を閉じても、あなたの開発は続きます。 ここからが、本当の始まりです。

あなたの最初のアプリが世に出る日を、心から応援しています。
